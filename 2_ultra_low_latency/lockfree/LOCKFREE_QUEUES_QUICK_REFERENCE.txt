┌──────────────────────────────────────────────────────────────────────────────────────┐
│ 1. SPSC (SINGLE PRODUCER SINGLE CONSUMER) - WAIT-FREE                               │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ Latency:    10-50 nanoseconds                                                        │
│ Guarantee:  Wait-free (every operation completes in bounded time)                    │
│ CAS Ops:    None (no contention)                                                     │
│ Best For:   Point-to-point, lowest latency                                           │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ USE CASES:                                                                           │
│  ✓ Market Data Feed → Strategy Engine          (most common)                        │
│  ✓ Strategy Engine → Order Gateway                                                   │
│  ✓ Order Gateway → Exchange Connectivity                                             │
│  ✓ Risk Check → Order Router                                                         │
│  ✓ Exchange Response → Order Manager                                                 │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ SAMPLE CODE:                                                                         │
│   SPSCRingBuffer<MarketDataTick, 8192> queue;                                        │
│                                                                                      │
│   // Producer (wait-free)                                                            │
│   while (!queue.push(tick)) { _mm_pause(); }                                         │
│                                                                                      │
│   // Consumer (wait-free)                                                            │
│   if (queue.pop(tick)) { process(tick); }                                            │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ PERFORMANCE:                                                                         │
│   50th %ile: 10-20 ns  │  99th %ile: 30-50 ns  │  Throughput: 50M ops/sec          │
└──────────────────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────────────┐
│ 2. MPSC (MULTI PRODUCER SINGLE CONSUMER) - LOCK-FREE                                │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ Latency:    50-100 nanoseconds                                                       │
│ Guarantee:  Lock-free (at least one producer makes progress)                         │
│ CAS Ops:    Producers use CAS, consumer doesn't                                      │
│ Best For:   Aggregating from multiple sources                                        │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ USE CASES:                                                                           │
│  ✓ Multiple Strategies → Single Order Router    (most common)                       │
│  ✓ Multiple Market Data Feeds → Consolidated Handler                                │
│  ✓ Multiple Risk Checks → Order Gateway                                             │
│  ✓ Multiple Exchange Responses → Order Manager                                      │
│  ✓ Multiple Algo Engines → Execution Gateway                                        │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ SAMPLE CODE:                                                                         │
│   MPSCRingBuffer<OrderEvent, 16384> queue;                                           │
│                                                                                      │
│   // Multiple Producers (lock-free)                                                  │
│   while (!queue.push(order)) { _mm_pause(); }                                        │
│                                                                                      │
│   // Single Consumer (wait-free)                                                     │
│   if (queue.pop(order)) { route(order); }                                            │
├──────────────────────────────────────────────────────────────────────────────────────┤

┌──────────────────────────────────────────────────────────────────────────────────────┐
│ 3. SPMC (SINGLE PRODUCER MULTI CONSUMER) - LOCK-FREE                                │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ Latency:    50-150 nanoseconds per consumer                                          │
│ Guarantee:  Lock-free (at least one consumer makes progress)                         │
│ Pattern:    BROADCAST (every consumer sees every message)                            │
│ Best For:   Broadcasting to multiple subscribers                                     │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ USE CASES:                                                                           │
│  ✓ Single Market Data Feed → Multiple Strategies (most common - broadcast)          │
│  ✓ Single Order Fill → Position + P&L + Risk (all need to know)                     │
│  ✓ Single Reference Data Update → All Strategies                                    │
│  ✓ Single Tick Feed → Multiple Analytics (VWAP, TWAP, Vol)                          │
│  ✓ Audit Trail: Single Event → Multiple Audit Systems                               │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ SAMPLE CODE:                                                                         │
│   SPMCRingBuffer<MarketDataTick, 8192, 8> queue;                                     │
│                                                                                      │
│   // Register consumers                                                              │
│   int consumer1_id = queue.register_consumer();                                      │
│   int consumer2_id = queue.register_consumer();                                      │
│                                                                                      │
│   // Single Producer (wait-free)                                                     │
│   while (!queue.push(tick)) { _mm_pause(); }                                         │
│                                                                                      │
│   // Multiple Consumers (lock-free, each sees every message)                         │
│   if (queue.pop(consumer1_id, tick)) { process1(tick); }                             │
│   if (queue.pop(consumer2_id, tick)) { process2(tick); }                             │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ PERFORMANCE (4 consumers):                                                           │

│ Guarantee:  Lock-free (system-wide progress)                                         │

│ Best For:   Load balancing, work pools                                               │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ USE CASES:                                                                           │
│  ✓ Multiple Order Sources → Multiple Risk Checkers (load balance)                   │

│  ✓ Event Bus: Any Component to Any Component (most flexible)                        │
│  ✓ Parallel Order Processing for horizontal scaling                                 │
├──────────────────────────────────────────────────────────────────────────────────────┤
│ SAMPLE CODE:                                                                         │
│   MPMCRingBuffer<OrderEvent, 16384> queue;                                           │

     echo 512 > /proc/sys/vm/nr_hugepages

╔══════════════════════════════════════════════════════════════════════════════════════╗
║                            COMMON PITFALLS                                           ║
╚══════════════════════════════════════════════════════════════════════════════════════╝

❌ Using std::queue + std::mutex
   → 100-500x slower than lock-free queues!
   → Use SPSC/MPSC/SPMC/MPMC instead

❌ Not pinning threads to CPU cores
   → Context switches destroy latency
   → Use taskset or pthread_setaffinity_np

❌ Using non-power-of-2 buffer sizes
   → Forces expensive modulo operations
   → Always use 1024, 2048, 4096, 8192, etc.

❌ Sharing cache lines between threads
   → False sharing causes cache thrashing
   → Use alignas(64) or CACHE_ALIGNED macro

❌ Not disabling hyper-threading
   → Non-deterministic latency from SMT
   → Disable for trading cores

❌ Using MPMC when SPSC would work
   → 10x latency penalty for no reason
   → Choose simplest queue that works

❌ Forgetting to use _mm_pause() in spin loops
   → Wastes power, lower performance
   → Always pause when spinning

❌ Not measuring latency with TSC
   → std::chrono too slow for nanosecond timing
   → Use __rdtsc() for cycle-accurate timing
